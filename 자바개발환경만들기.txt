1970 C (데니스 리치) : 서로 다른 기종의 컴퓨터에서도 같은 언어로 프로그래밍이 가능하도록 함.
1980 C++ (비야네 스트로스트룹) : C언어의 기본을 유지하면서 객체 지향의 개념을 도입.
1990 JAVA (제임스 아서 고슬링) : 메모리 관리, 포인터 삭제, 서로 다른 hw/sw에서 똑같이 실행되는 프로그램

JRE (Java Runtime Environment, 자바 실행 환경)
- 자바로 개발된 프로그램을 실행하기 위해 필요한 요소
- 마인크래프트를 실행하기 위해서는 자바가 설치되어 있어야 한다.(이 때 자바는 JRE를 말함.)

JDK (Java Development Kit, 자바 개발 도구)
- 자바 언어로 프로그램을 개발하기 위한 도구 모음, 실행 및 테스트를 위해 JRE를 포함한다.
- 자바 기반의 안드로이드 앱을 개발하기 위해서 자바를 설치해야 한다. (이 때 자바는 JDK를 말함.)
- 기본적으로 JRE가 포함되어 있다.

설치 파일을 더블클릭해서 실행하고 설치한 이후
win + r(실행) -> cmd 입력

[*.java] : souce file, 자바 문법에 맞게 프로그램 코드가 사람에 의해 작성된 파일

[*.class] : byte code, 소스 코드를 자바 컴파일러가 변환해서 자바 가상머신에서 실행할 수 있도록 만든 파일.

=> 매번 메모장과 cmd를 이용해서 프로그램 개발하기에는 번거롭기 때문에
소스코드 작성, 컴파일, 실행, 프로젝트 및 파일 관리를 도와주는 추가 프로그램이 필요하다.

<자바 환경 변수 설정>
환경 변수 : 운영체제에서 자주 사용하는 값을 변수로 만들어서 저장해두었다가 필요할 때 참조하거나, 다른 프로그램이 사용하는 값.
변수 : 담을 수 있는 그릇, RAM에 위치한다. ex) 쇠숟가락
PATH : 환경 변수의 일종, 자주 사용하는 경로 (폴더 or 디렉토리)를 등록하는 곳.
          ㄴ> PATH에 등록된 경로는 컴퓨터 어디에서나 접근할 수 있게 된다.
win + r (실행) -> sysdm.cpl -> 고급 -> 환경 변수 -> 시스템 변수

JAVA_HOME	(자바가 설치된 폴더 경로)
PATH		요소를 삭제하지 않고, 기존 요소에%JAVA_HOME%\bin 을 추가하기

※ 환경변수 설정 이후, 확인을 눌러서 창을 닫아야 환경변수가 적용됨,
이미 실행했던 프로그램은 환경변수가 적용되지 않아서 재실행해야 환경변수가 적용된다.

※ 파일명 앞에 .이 붙어있으면 중요하니까 건들지 않는 파일 (리눅스 환경에선느 숨겨진 파일)

JRE : 자바 실행 환경, 자바 가상머신을  포함한다. 자바로 작성된 프로그램을 실행하기 위해서 필요.
JDK : 자바 개발 도구, 자바를 이용하여 프로그램을 개발 할 수 있다. 자바 컴파일러가 포함됨.
source 혹은 src, *, .java, 소스 파일 : 사람이 자바 문법으로 작성한 프로그램 소스 코드, 컴파일하면 바이트코드로 변환됨.
byte code, bin, *, .class, 바이트 코드 파일 : 컴파일러를 거쳐 소스코드가 변환된 형태, JVM위에서 실행할 수 있다.
JVM (Java Virtual Machine) 자바 가상 머신 : 자바 프로그램을 실해하기 위해 OS 위에서 실행중인 프로그램 
IDE : 통합 개발 환경, 소스 작성, 파일 관리, 컴파일 및 인터프리트, 자동 완성, 디버깅, 실행을 하나의 프로그램에서 지원한다.
